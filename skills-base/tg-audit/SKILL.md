# tg-audit — Аудит Telegram-канала

Анализирую упаковку TG-канала: bio, закреп, название, профиль.
Результат — живой разбор голосом Олега. Не GPT-отчёт, не корпоративная простыня.

## Промпты

| Файл | Когда использовать |
|------|-------------------|
| [prompts/p1_bio.md](prompts/p1_bio.md) | Аудит только bio и имени профиля |
| [prompts/p2_channel.md](prompts/p2_channel.md) | Полный аудит канала (название + bio + закреп + профиль) |
| [prompts/p3_antiai.md](prompts/p3_antiai.md) | Рерайт: убить AI-стиль, добавить живость |
| [prompts/p5_oleg_voice.md](prompts/p5_oleg_voice.md) | Голос Олега — финальная прогонка через стиль Лаврентьева |
| [prompts/p6_clean.md](prompts/p6_clean.md) | Очистить текст от символьного мусора |

## Как работать — ДВА РЕЖИМА

### Режим 1: Рерайт закрепа / текста (только текст, без структурного аудита)
1. Применить p3_antiai.md — БЫЛО/Проблема/СТАЛО/Почему лучше по каждому блоку
2. **ВСЕГДА** → прогнать финал через p5_oleg_voice.md

### Режим 2: Полный аудит канала (название + bio + закреп + профиль)
1. Применить p2_channel.md — структурный разбор по блокам с вариантами
2. **ВСЕГДА** → прогнать финальный комментарий через p5_oleg_voice.md

p3_antiai — для РЕРАЙТА текста. p2_channel — для СТРУКТУРНОГО АУДИТА. Не путать.

## Типовые сценарии

- **Только bio** → p1_bio.md → p5_oleg_voice
- **Полный аудит канала** → p2_channel.md → p5_oleg_voice
- **Аудит закрепа с рерайтом** → p3_antiai → p5_oleg_voice
- **Готовый текст звучит роботом** → p3_antiai → p5_oleg_voice
- **Текст с символами/мусором** → p6_clean.md → p5_oleg_voice

## Формат вывода — файл

Результат каждого аудита сохранять в папку:
`/root/YandexSync/ClaudeCode/02_club/audits/`

Имя файла: `audit_ИМЯ_ДАТА.md` — например:
- `audit_svetlana_2026-05-25.md`
- `audit_marina_2026-05-26.md`

Имя = имя автора или @username канала (строчными, без пробелов). Дата обязательна — аудиты накапливаются для мегаразбора и статьи.

### Структура файла — Режим 1 (рерайт):
1. Разбор по блокам (БЫЛО / Проблема / СТАЛО / Почему лучше)
2. Итоговая таблица оценок (Было X/10 → Стало Y/10) по 5 критериям
3. Итоговый текст (чистовик в блоке кода)
4. Финал голосом Олега с P.S.

### Структура файла — Режим 2 (полный аудит):
1. Диагноз одной строкой
2. Блоки: Название / Bio / Закреп / Профиль
   - Каждый блок: Оценка X/10 + Что сломано + Что исправить + Варианты
   - Закреп: разбор по абзацам (цитата + объяснение 2-4 строки) + орфография + слабые места + короткая версия ≤800 зн. + расширенная ≤1800 зн. + оценки (Читаемость/Удержание/Сторителлинг/Эмоции/Продажи) + визуал
3. План правок за 30 минут (3 шага с временем)
4. **Итоговая оценка | Олег Лаврентьев** — ОБЯЗАТЕЛЬНЫЙ БЛОК:
   - Канал + дата + Общая оценка X/10
   - Вердикт одной строкой
   - Что не работает (приоритет по порядку, 3-5 пунктов)
   - Что уже работает — не трогать
   - Главный совет (одно действие)
   - Потенциал канала (2-3 предложения)
5. Финал голосом Олега с P.S. (живой разговорный разбор)

## Эмодзи в аудите

Разбор по блокам, таблица оценок, чистовик — **без эмодзи**.

Эмодзи только в финале голосом Олега — немного, по делу, там где живой человек их бы поставил. Не для украшения — для ощущения что пишет человек, а не отчёт.

## Как звучит правильный результат

Не "данный канал имеет потенциал для роста" — это смерть.
А "короче, закреп написан как резюме на Hh.ru. Никто не читает резюме в Telegram."

Если после аудита можно поставить подпись GPT — переделать.
