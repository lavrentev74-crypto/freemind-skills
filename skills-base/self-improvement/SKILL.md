# Self-Improvement — Контролируемое самоулучшение

## Принцип

Не каждая ошибка → в ядро. Сначала наблюдение → урок → паттерн → и только потом правило ядра.

## 4-уровневая лестница

```
Событие → Урок → Паттерн → Правило ядра
```

| Уровень | Когда | Где хранить |
|---------|-------|-------------|
| **Событие** | Произошла ошибка или удачный ход | — (в голове) |
| **Урок** | Есть локальный вывод что и почему | `memory/lessons.md` |
| **Паттерн** | Урок повторился 2+ раза ИЛИ высокая критичность | `memory/patterns.md` |
| **Правило ядра** | Паттерн устойчив, короток, безопасен, применим широко | CLAUDE.md / AGENTS.md |

## Когда записывать урок

Записывай если:
- агент повторил уже исправленную ошибку
- агент потерял важный контекст
- агент сработал особенно удачно (самоулучшение — не только от провалов)
- сбой оказался следствием архитектурной слабости

Не записывай:
- разовые оговорки без системного смысла
- временные хаки и костыли
- вкусовые предпочтения без повторного подтверждения

## Шаблон урока (lessons.md)

```md
## YYYY-MM-DD | Название урока

### Ситуация
[что произошло]

### Причина
[почему случилось]

### Вывод
[как действовать лучше]

### Статус
- уровень: lesson
- подтверждено повторно: нет / да
- поднять в pattern: нет / да
```

## Правило продвижения

**Lesson → Pattern** если:
- ситуация повторилась 2+ раза
- ошибка критичная даже при одном повторе
- правило переносимо на другие задачи

**Pattern → Ядро** если:
- доказал пользу на нескольких кейсах
- не конфликтует с безопасностью
- нужен почти всегда, не в редких случаях
- короткий и однозначный

## Запрет на тихое изменение ядра

Если тема касается — **сначала показать владельцу**:
- безопасность и ключи API
- удаление данных
- деньги и оплаты
- публичный тон и бренд
- системные cron/background задачи

## Откат паттерна

Если паттерн попал в ядро и начал вредить:
1. Убрать из ядра
2. Опустить обратно в `memory/patterns.md`
3. Пометить как `rollback: да` с причиной
4. Записать причину в дневную память

## Частые ошибки

- **Учится на мусоре** → писать в lessons только повторяемое или критичное
- **Паттернов слишком много** → не меньше 2 подтверждений или 1 высокая критичность
- **Ядро разрослось** → в ядре только короткие, универсальные, часто применимые правила
