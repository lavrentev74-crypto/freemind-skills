# Context Lifecycle — Дисциплина контекста

## Три сущности (не путать)

| Сущность | Что это |
|----------|---------|
| **Контекст** | Рабочее окно прямо сейчас — думаем и действуем здесь |
| **Память** | Долгосрочное хранилище: LightRAG, MEMORY.md, daily notes |
| **Handoff** | Структурированный мост между сессиями |

Если всё хранить в контексте — агент перегреется. Если не делать handoff — следующая сессия начнёт с нуля.

## Пороги контекста

| Заполнение | Действие |
|-----------|---------|
| 0–55% | Работаем спокойно, не копируем лишнего |
| 55–70% | Сжимаем шум, убираем второстепенное из окна |
| 70–85% | Собираем суть, готовим handoff **заранее** |
| 85%+ | Стоп. Записать handoff → завершить контур → reset |

## Фазы работы в длинной задаче

1. **Старт** — взять только нужные файлы, не тащить всё подряд
2. **Работа** — держать в окне только то, что помогает следующему шагу
3. **Подготовка handoff** (70%+) — решения, незавершённое, изменённые файлы
4. **Handoff** — записать до деградации, не ждать перегрева
5. **Reset** — новый контур читает handoff, поднимает только нужную память

## Шаблон handoff

```md
# Handoff | YYYY-MM-DD HH:MM

## Тема
[1-2 предложения о чём работа]

## Решения
- [что и почему]

## TODO (незавершённое)
- [ ] [задача]

## Изменённые файлы
- [путь] — [что изменено]

## Контекст для продолжения
[суть без мусора, без ссылок на номера сообщений]
```

## Что НЕ писать в handoff

- Весь чат подряд
- Пароли, токены, ключи API
- Мелкие промежуточные рассуждения
- Всё что уже есть в постоянной памяти

## После compaction / новая сессия

1. Прочитать `memory/handoff.md`
2. Поднять только нужные слои памяти
3. Восстановить: тема → решения → незавершённое → файлы
4. Продолжать работу (не восстанавливать весь старый шум)

## Частые ошибки

- **Handoff слишком поздно** → начинай в зоне 70–85%, не на 85%+
- **MEMORY.md как свалка** → держи коротким, только устойчивые факты недели/месяцы
- **После reset грузишь старый мусор** → поднимай только handoff + строго нужная память
