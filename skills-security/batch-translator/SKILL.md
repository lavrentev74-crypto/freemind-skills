# Batch Translator

Пакетный перевод файлов с сохранением структуры.

## Правила перевода

**Переводить:**
- Весь текст на исходном языке → целевой язык
- Заголовки `#`, `##`, `###`
- Содержимое таблиц (заголовки и ячейки)
- Комментарии в коде (если явно просят)

**НЕ переводить:**
- Код внутри ` ```code blocks``` ` и инлайн `` `code` ``
- XML/HTML теги (`<context>`, `<instructions>`, `<example>`)
- YAML frontmatter (`---` блоки)
- Ссылки: `[текст](url)` — только текст, не URL
- Технические термины без устоявшегося перевода: prompt, token, LLM, RAG, CoT, API, webhook, endpoint
- Имена файлов и пути

## Workflow

### 1. Получить список файлов
```bash
ls /путь/к/папке/
# или рекурсивно
find /путь -name "*.md" -type f
```

### 2. Для каждого файла
1. Прочитать файл (Read)
2. Перевести соблюдая правила выше
3. Записать обратно (Write/Edit) — перезаписать тот же файл
4. Перейти к следующему

### 3. Делегировать агенту для больших папок
Если файлов > 5 — запустить Agent в фоне:
```
Agent(
  prompt="Переведи все .md файлы в /папка/ на русский.
  Правила: код не трогать, YAML не трогать, теги не трогать.
  Файл за файлом: читай → переводи → записывай.",
  run_in_background=True
)
```

## Типичные команды

**Перевести один файл:**
> "Переведи /path/file.md на русский"

**Перевести папку:**
> "Переведи все файлы в _inbox/skill-name/ на русский, сохрани структуру"

**Перевести скилл из ~/.claude/skills/:**
> "Скопируй скилл X в _inbox/ и переведи на русский"

## Поддерживаемые форматы
- `.md` — Markdown (основной)
- `.txt` — простой текст
- `.json` — только значения строк, не ключи
- `.yaml` / `.yml` — только значения, не ключи

## Качество

- Технические термины: оставлять английские если нет устоявшегося русского
- Стиль: профессиональный, без машинного привкуса
- После перевода: проверить что код-блоки не тронуты
